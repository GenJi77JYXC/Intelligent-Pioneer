# 目录结构说明

```bash
intelligent-pioneer/
├── docs/
│   ├── arch/
│   │   ├── 01_overall_architecture.md  # 总体架构设计
│   │   └── 02_database_design.md       # 数据库模型设计
│   │
│   ├── api/
│   │   └── openapi.yaml                # OpenAPI (Swagger) 规范文件
│   │
│   ├── development/
│   │   ├── 01_getting_started.md       # 本地开发环境搭建指南
│   │   └── 02_coding_style.md          # 编码规范
│   │
│   ├── deployment/
│   │   ├── 01_production_setup.md      # 生产环境部署手册
│   │   └── 02_configuration.md         # 生产环境配置项说明
│   │
│   ├── user_guide/
│   │   ├── 01_agent_installation.md    # Agent 安装和注册指南
│   │   └── 02_kb_management.md         # 如何创建和管理知识库
│   │
│   └── images/                         # 存放文档中引用的图片
│       └── architecture_diagram.png
│
├── cmd/
├── internal/
└── ... (其他项目目录)
```



**目录结构解释:**

- **arch/ (Architecture - 架构设计):**
  - 存放系统的高层设计文档。
  - 01_overall_architecture.md: 描述项目的总体架构、核心组件（Agent, Backend, Platform）、它们之间的交互方式，可以把你之前画的架构图放进去。
  - 02_database_design.md: 详细说明 PostgreSQL 的表结构、字段含义，以及 Elasticsearch 的索引设计。
- **api/ (API 文档):**
  - openapi.yaml: 强烈推荐使用 [OpenAPI (以前叫 Swagger)](https://www.google.com/url?sa=E&q=https%3A%2F%2Fswagger.io%2Fspecification%2F) 规范来定义你的所有 API。它可以被各种工具用来自动生成交互式 API 文档、客户端代码甚至服务器桩代码。
- **development/ (开发指南):**
  - 01_getting_started.md: 比 README.md 更详细的开发环境搭建指南，可以包含如何调试、如何运行测试等内容。
  - 02_coding_style.md: 团队的编码规范，例如 Go 的命名约定、错误处理方式、日志规范等。
- **deployment/ (部署运维):**
  - 01_production_setup.md: 如何将项目部署到生产服务器的详细步骤，包括服务器要求、依赖安装、安全配置等。
  - 02_configuration.md: 详细解释 config.yaml 中每一个配置项的含义和可选值。
- **user_guide/ (用户手册):**
  - 面向项目最终用户的文档。
  - 01_agent_installation.md: 指导运维人员如何在目标机器上安装和注册 intelligent-pioneer 的 Agent。
  - 02_kb_management.md: 指导运维专家如何在系统中创建和管理知识库条目。
- **images/:**
  - 一个专门存放所有 Markdown 文件中引用的图片的地方，保持根目录整洁。
